1. 今日のゴールと3つの型

自分の手で、ショート動画を実際に1本書き出すところまで。 作り方の“3つの型”を、手を動かして体験します。

むずかしい契約や公開設定はしません。まずは「自分のPCで動画ファイル（縦型 9:16 の short.mp4）ができる」ところがゴールです。投稿（YouTube連携）は、やりたい人だけ最後に。

① 静止画ショート

AIイラスト＋ナレーション。Gemini「nanobanana」で絵を生成して並べる。

アニメ調・偉人伝・雑学

② 素材ショート

ストック動画＋ナレーション。Pexels/Pixabay/Envato の映像を組み合わせる。

解説・TOP5・ニュース

③ 切り抜きショート

長い動画から名場面を縦型に。文字起こし→いいとこ抽出→字幕。

対談・講義・Q&A

どの型も、流れは同じ「Skillを呼ぶ → 構成を確認 → 生成 → 自分の目でチェック →（必要なら）投稿」です。違うのは素材をどこから持ってくるかだけ。

時間の目安内容
0:15最低限の地図（Claude Codeの言葉と仕組み）
0:15準備（アプリ・APIキー・IrodoriTTS起動）
0:45ハンズオン①：静止画ショート（全員でしっかり）
0:30ハンズオン②：素材ショート
0:30ハンズオン③：切り抜きショート
0:15ブラウザ自動化・まとめ・質問

正直な注意：3時間に全部を“完璧に”は入りません。①を全員で完走 →②③は要点デモ＋各自トライ、が現実的です。①で「作れた！」を持ち帰るのが最優先。

2. 最低限の地図（Claude Codeの言葉）

用語は「使う直前に1個ずつ」。それぞれ、ひとことの言い換え＋たとえで覚えれば十分です。

Claude Code って何？

LLMネットの文章を大量に読んだ“物知りな助手”
Claudeその助手のひとり（Anthropic社製）
Claude Codeその助手に“手”を付けたもの

ふつうのAIチャットは「答えを言う」だけ。Claude Code は、あなたのPCを実際に操作して、ファイルを作ったり、動画を書き出したりできます。素材集め→音声生成→編集→書き出しまで、通しで手を動かしてくれるのがポイント。

覚えておきたい4つの言葉

言葉ひとことでたとえ
Context windowいま同時に覚えていられる量作業机の広さ。埋まると古い話を忘れる
Agentic loop考える→やる→確認→直す を自分で繰り返す自走する職人。だから「止める・確認」が大事
MCP外部サービスに挿すコネクタAI用のUSB端子。素材サイトやブラウザに挿す
Skillsよく使う作業手順書のパックレシピ集。/ で呼び出せる
Context window のコツ：話が長くなって反応が変だと思ったら、新しいセッションで「今やりたいこと」だけ伝え直す。机を片付ける感覚です。

おまけ：Agent / サブエージェント という言葉も出てきます。大きな作業を別の“担当者”に分けて並行で進める仕組み。私たちは結果を受け取るだけでOK。Skillsの中で自動的に使われることがあります。

得意なこと・苦手なこと

得意：素材集め・音声生成・編集の自動化／決まった手順のくり返し／試作を速く／「ちょっと直して」を何度でも。

苦手：あいまいな丸投げ、細かい“感性の最終調整”、最新すぎる情報、権利関係の自動判断。→ 具体的に頼む・できた素材は自分の目で確認するのが大事。特に著作権／利用規約は人間が最終チェック。

3. 準備（道具とカギ＝APIキー）

道具（Claude Code・Remotion・IrodoriTTS・素材取得）はプロジェクトに最初から入っています。みなさんがやるのは「動かすカギ（APIキー）」を入れることと、音声エンジンを起動することだけ。

APIキーとは

APIキー外部サービスを使うための“合言葉” もらった合言葉を .env（環境設定ファイル）に貼り付けておくと、あとはClaudeが自動で使ってくれます。

合言葉＝あなたの財布。人に見せない・SNSに貼らない・GitHubに上げない。（.env は最初からアップロード除外に設定済みです。）

用途別・今日そろえるカギ

用途カギ / 道具メモ
🎨 画像生成（①）GEMINI_API_KEYGemini 2.5 Flash Image（通称 nanobanana）。取得：aistudio.google.com/apikey
🎞️ 動画素材（②）Pexels / Pixabay（任意・無料）Envato は有料で高品質。素材取得に使用
🗣️ 音声（①②）IrodoriTTSローカルで動く。無料・回数制限なし。キー不要、起動するだけ
📝 文字起こし（③）OPENAI_API_KEY切り抜きの文字起こし（Whisper）で使用
⬆️ 投稿（任意）YouTube OAuth投稿したい人だけ。最初の1回だけブラウザで許可
詰まっても大丈夫。カギが1つ（GEMINI_API_KEY）入れば①は動きます。まず①を全員で完走しましょう。

IrodoriTTS（ナレーション音声）

IrodoriTTS文章を“しゃべり声”に変える、手元で動く音声エンジン

  • 無料・回数制限なし（自分のPCの中で動くので、クラウドの料金がかからない）
  • 声の雰囲気を言葉で指定できる（例：「明るく聞き取りやすい男性の語り口で、好奇心と驚きを込めて」）
  • 参照音声を使うと、動画ごとに声がブレない（同じ声で統一）
  • 読み間違いは、台本のその語をひらがなに直せば直る（例：後の世 → のちの世）

使う前に一度だけ「起動」しておけばOK。あとはClaudeが各セリフを自動で音声化します。起動の合言葉が分からなければ、Claudeに「IrodoriTTSを起動して」と頼めば手順を案内してくれます。

「①のテーマを伝えて、構成案が返ってくる」ところまで来たら準備OK。

4. ショート動画ができるまで（共通の流れ）

どの型でも、工程は同じ4つです。

  1. 台本：何を話すか（構成・ナレーション文）
  2. 素材：絵や映像（①AI生成 ／ ②ストック ／ ③元動画から切り出し）
  3. 音声：ナレーション（IrodoriTTS）
  4. 合成：1本の動画に（Remotion / ffmpeg）。字幕・テロップ・BGMも自動

Remotion“コードで動画を組み立てる”仕組み。編集ソフトの代わり タイムラインを手で並べる代わりに、設定（テキスト）から動画を生成します。同じ型で量産でき、やり直しが一瞬。コードを書くのはClaudeで、みなさんは書きません。「編集ソフトを自動操作してくれる人がいる」くらいの感覚でOK。

5. ハンズオン①：静止画ショート

AIイラスト＋ナレーション。使うSkillは youtube-shorts（アニメ調・静止画）。「アニメ調で作って」と言えば呼ばれます。

仕組み：構成案（6コマの台本）→ nanobanana で画像生成（1コマ1枚・縦型）→ IrodoriTTS で音声 → 字幕＋BGMで合成 → 9:16 の short.mp4。画像は1枚あたり数円、6枚で十数円ほど。

STEP 1テーマを伝える

アニメ調の静止画ショートを作りたい。テーマは「スティーブ・ジョブズが風呂に入らなかった話」。 まず6コマ分の構成（各コマの場面＋ナレーション文）を提案して。

STEP 2構成案を直す（ここが一番ラク）

絵を作る前に台本を固めます。「3コマ目のナレーションをもっと短く」「オチをはっきりさせて」など、言葉で直します。

コマ割りとナレーションに納得できたか。ここで固めるほど後がスムーズ。

STEP 3「OK」で生成 → 確認

「この構成でOK。画像と音声を作って動画にして」と伝えると、画像→音声→合成まで自動で進みます。出てきた short.mp4 を必ず自分の目で再生。

STEP 4気になる所だけ直す

  • 絵が気に入らない →「3番だけ作り直して。もっと夜の雰囲気で」
  • 声が硬い →「ナレーションをもっと明るい語り口に」
  • 読み間違い → 台本のその語をひらがなに直す
実在の人物・ブランドのAI画像は“そっくり”を避け、雰囲気で。投稿時は各プラットフォームのルールも確認しましょう。

①の実例（クリックで再生）

1990円フリースで社会現象を起こした男 ▶ 8,832 回 刑務所で1000冊読んだ男 ▶ 5,198 回 カモノハシは、生物学の常識を全部破る動物 ▶ 5,104 回

アニメ調の静止画＋ナレーション。出典：claude-code-gen-shorts

6. ハンズオン②：素材ショート

ストック動画＋ナレーション。使うSkillは stock-video-shorts（このリポジトリのおすすめ既定）。型は2つ：concept-explainer（1テーマ解説）と remotion-toplist（TOP5ランキング）。

仕組み：構成案 → Pexels等から映像を英語キーワードで自動取得 → IrodoriTTS で音声 → Remotion で字幕・アイコン・テロップ込みに合成 → 縦型 short.mp4。

STEP 1テーマ＋型を伝える

ストック動画で TOP5 ショートを作って。テーマは「世界一大きい動物 TOP5」。 まず構成案（各順位の名前・数値・ナレーション・映像の検索ワード）を出して。

STEP 2構成を確認 → 生成

順位・数値・ナレーション・検索ワードを確認して「OK」。ストック動画の取得→音声→Remotion合成まで自動で進みます。

STEP 3確認・差し替え

「3位の映像が地味」→ 検索キーワードを変えて差し替え。「タイトルの色を変えて」なども言葉でOK。

縦型 short.mp4 を再生して、映像と字幕・ナレーションが合っているか。

②の強み：著作権と量産

  • 素材は正規のストックサービスから取得 → 商用利用の見通しが立てやすい
  • コストがほぼ定額（無料API or 月額）→ 本数を増やしても安い
  • 同じ型で横展開しやすい（TOP5を別テーマで量産）
ライセンスは型ごと・素材ごとに違います。Pexels / Pixabay は寛容ですが、最終確認は人間。特に収益化・商用で使うなら、各サービスの利用規約をその都度チェック。

②の実例（クリックで再生）

外国人が驚く日本の技術 TOP5 ▶ 6,657 回 毒が強い動物 TOP5、1位はハコクラゲ ▶ 4,506 回 世界一大きい動物 TOP5（プロンプト例と同じ） ▶ 3,469 回

ストック実写＋アイコン＋字幕アニメ。出典：claude-code-gen-shorts

7. ハンズオン③：切り抜きショート

長い動画から“名場面”を縦型に。使うSkillは youtube-clipper（別リポジトリ claude-code-clip-video）。対談・講義・Q&A に向きます。

仕組み：元動画をダウンロード → Whisper で文字起こし（秒単位の字幕データ）→ ★Claude本人が文字起こしを読んで40〜60秒の名場面を選ぶ → 顔寄せ＋字幕で 9:16 に書き出し。

ここが面白いポイント：“バズりスコア”のような自動採点ではありません。Claudeが「最初の2秒でフックがある／話が一つで完結している」という基準で、選んだ理由を説明しながら切り抜きを提案します。

STEP 1元動画を渡す

この動画 https://youtu.be/… から、40〜60秒の切り抜きショートを3本作りたい。 文字起こしして、フックの強い場面を「選んだ理由」つきで提案して。

STEP 2候補を選ぶ → 書き出し

提案された候補から採用するものを選ぶと、縦型・顔寄せ・字幕入りで書き出されます。自動で付く処理：

  • カラオケ字幕：今しゃべっている言葉が黄色くハイライト
  • 顔追従＋ズーム：話者の顔を中央に寄せて縦型にトリミング
  • 間（ま）のカット：無音や「えーと・あの」を自動で詰められる
  • 結論先出し：オチを冒頭に持ってくる構成も指定できる

STEP 3確認してから公開

出力は最初非公開で書き出し → short.mp4 を確認してから公開判断、が安全です。

権利の大前提：他人の動画を切り抜くには許諾が必要です。自分の動画、または許可・契約のある素材で行いましょう。

③の実例（クリックで再生）

ひろゆき「AIでエンジニアは失業しません」 ▶ 5,623 回 ひろゆき「子供のゲーム課金、親はこうしろ」 ▶ 4,452 回 ひろゆき「今の少子化問題の正体」 ▶ 3,838 回

長い対談から名場面を縦型＋字幕に。出典：claude-code-clip-video

補足：このSkillには話者タイプ別の派生（clipper-explainer＝一人語り・講義向け、clipper-qa＝Q&A・質問コーナー向け）もあります。文字起こしには OPENAI_API_KEY が必要です。

8. おまけ：ブラウザ自動化

Agent browserAIが人の代わりにブラウザを開いて操作する仕組み 画面を見てクリック・入力・スクロール・スクリーンショットができます。Claude Code に“ブラウザの手”を足すイメージ（MCPの一種。Claude in Chrome 拡張や Playwright MCP など）。

動画制作での使い道

  • ネタ・トレンド集め：急上昇やコメントを一覧化（「このページの上位10件を表に」）
  • 素材さがし：素材サイトを横断して候補を収集
  • 投稿まわりの下ごしらえ：タイトル・説明文の下書き入力（最終送信は人がチェック）
マナー：少量・学習目的で。相手サイトの規約・回数制限に配慮（大量取得しない）。自動“送信・投稿”は事故のもと →下書きまで＝人が最後に確認が安全です。

9. コピペ用プロンプト集

そのまま貼って使えます。「」の中身を自分のテーマに変えるだけ。

① 静止画ショート

アニメ調の静止画ショートを作りたい。テーマは「__ここにテーマ__」。 まず6コマ分の構成（各コマの場面＋ナレーション文）を提案して。 OKを出したら、画像と音声を作って 9:16 の動画にまとめて。

② 素材ショート（TOP5）

ストック動画で TOP5 ショートを作って。テーマは「__ここにテーマ__ TOP5」。 まず構成案（各順位の名前・数値・ナレーション・映像の検索ワード）を出して。 確認後、素材取得から Remotion 合成まで進めて。

② 素材ショート（1テーマ解説）

ストック動画で「__ここにテーマ__」を30秒で解説するショートを作って。 concept-explainer の型で、シーン構成・ナレーション・映像の検索ワードをまず提案して。

③ 切り抜きショート

この動画 __URL__ から、40〜60秒の切り抜きショートを3本作りたい。 文字起こしして、フックの強い場面を「選んだ理由」つきで提案して。 採用したものは、顔寄せ＋字幕つきの縦型で、まず非公開として書き出して。

直すときの言い回し

  • 「3番だけ作り直して。もっと__こんな雰囲気__で」
  • 「ナレーションをもっと__明るい/落ち着いた__語り口に」
  • 「__この語__をひらがなにして読み間違いを直して」
  • 「いったん止めて。今の構成だけ見せて」
